# Hegemony v0.1 Rules Spec

This is the implementation contract for the first browser prototype. It deliberately excludes many systems from the archive rules so the game can become playable before the design is complete.

## Goal

Build a hotseat browser prototype with a real rules engine and a small local turn controller. The prototype should prove the core loop:

1. Place capital.
2. Place colony.
3. Start season.
4. Collect income.
5. Build basic buildings.
6. End turn.

Online multiplayer, events, assembly voting, national ideas, luxury goods, and final victory conditions are deferred.

## Technology

- TypeScript for rules and UI.
- React plus Vite for the browser client.
- Local React state for the v0.1 hotseat turn controller.
- SVG for the first map renderer.

## Player Count

v0.1 supports 4 seats in hotseat mode.

The game should be written so 2 to 4 players can be supported later without rewriting the rules.

## Map

v0.1 uses one inland hex board:

- Radius 3 axial hex grid.
- 37 terrain tiles.
- No coastal tiles yet.
- No ports yet.
- No luxury goods yet.

The terrain deck uses the PDF's counts:

- 6 mountain.
- 9 hill.
- 14 forest.
- 8 plains.

The hill distribution is a temporary completion because the PDF does not fully specify all 9 hill tiles.

## Phases

### setupCapital

Each player places one capital.

Rules:

- Must place on an empty tile.
- Cannot place adjacent to another city or capital.
- One move ends the player's setup turn.

### setupColony

Each player places one colony.

Rules:

- Must place within 1 hex of a friendly settlement.
- Cannot place on a tile with a city or capital.
- Cannot place adjacent to an enemy capital.
- A tile may contain at most 2 colonies.
- A player cannot have two colonies on the same tile.
- One move ends the player's setup turn.

### gameplay

Each player's turn:

1. Collect income once.
2. Optionally build basic buildings.
3. End turn.

Season starts before player 1's turn. In v0.1, a new season is detected when play returns to player 1 after at least one player has collected income.

## Settlements

### Capital

v0.1 capital rules:

- Counts as a city.
- Collects double tile resource income.
- Has 20 pop capacity.
- Has terrain building slots plus 4.
- Starts with 1 citizen, 2 freemen, and 3 slaves.

### Colony

v0.1 colony rules:

- Has 4 pop capacity.
- Has no building slots.
- Can share a tile with one enemy colony.
- Shared colony tile income is halved for both players.
- Starts with 3 slaves.

Colony upgrade is deferred.

### City

Non-capital cities are deferred until colony upgrade is implemented.

Target city rule for later:

- 10 pop capacity.
- Terrain building slots plus 2.
- Cannot exist adjacent to any other city.

## Resources

Tracked resources:

- Wood.
- Stone.
- Gold.
- Food.
- Influence.
- Unrest.

Starting resources in v0.1:

- 20 wood.
- 10 stone.
- 10 gold.
- 12 food.
- 0 influence.
- 0 unrest.

These are generous because v0.1 needs fast building tests.

## Population

Tracked pop types:

- Citizens.
- Freemen.
- Slaves.

v0.1 uses the tentative conversion:

- 1 citizen = 2 freemen = 3 slaves.

This conversion is not yet used by many systems because event cards are deferred.

Income:

- Each citizen gives +1 influence and -2 food.
- Each freeman gives +2 gold and -1 food.
- Every 3 slaves gives +1 of the tile's primary resource and -1 food.
- Every 9 slaves gives +1 unrest.

Food shortage:

- If calculated food income is negative, add that absolute value to unrest.
- The two-consecutive-turn starvation rule is documented but deferred.

## Buildings

v0.1 implements only the basic buildings needed to test city growth.

### Marketplace

Cost:

- 12 wood.

Effect:

- Adds 1 freeman.

### Temple

Cost:

- 6 stone.

Effect:

- Adds 1 citizen.

### Workshop

Cost:

- 12 wood.

Effect:

- Adds 2 slaves.

### Granary

Cost:

- 12 wood.
- 2 stone.

Effect:

- Adds +2 food income.

## Deferred Systems

The following systems are intentionally excluded from v0.1:

- Coastal tiles.
- Ports.
- Luxury goods.
- Founding new colonies after setup.
- Upgrading colonies into cities.
- Player event cards.
- Public event cards.
- Resolution cards.
- Hegemony Assembly.
- Politicians.
- National ideas.
- Trade.
- Repeal and veto.
- Victory conditions.
- AI or bots.
- Persistent accounts.
- Remote multiplayer deployment.

## Implementation Rules

The rules engine should stay separate from the React UI.

The React UI may:

- Render state.
- Let players select actions.
- Show legal/illegal feedback.
- Call shared rules functions through the local turn controller.

The React UI must not:

- Own permanent game state.
- Apply resource changes directly.
- Decide whether an action is legal.
- Duplicate costs or production formulas outside shared data.

## Next Milestone After v0.1

v0.2 should add:

- Found colony action.
- Upgrade colony to city.
- City adjacency validation.
- Basic pop capacity and overcapacity unrest.
- A temporary 10-season scoring condition.
