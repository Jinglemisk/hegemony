Hegemony Project Overview

This page is the living map of the Hegemony project: current status, roadmap, core files, design explorations, implemented systems, and deferred work. Update this document as the project changes.

Project Roadmap

Phase 0: Scaffolding and Rules Capture Project setup, docs, first rules translation, and removal of boardgame.io.
Phase 1: Core Playable Prototype Map, setup flow, turns, income, expansion, city upgrades, pops, buildings, holdings UI, and basic local play.
Phase 2: Rule Depth Unrest events, food pressure, scoring pressure, assembly voting, richer building effects, and better balance rules.
Phase 3: Multiplayer Foundation Server state, rooms, reconnects, turn validation, persistence, and lobby flow.
Phase 4: Content and Polish National ideas, luxury goods, victory conditions, presentation overhaul, and UX refinement.

Brand and Art Direction

The playable UI now uses the Codex Banana light-mode asset set directly from assets/codex-banana-showcase/. Those copied PNG atlases are the durable source for the app chrome, icons, terrain art, and hero backdrop; the showcase pages are only visual reference artifacts.

Implemented Now

  • 37-tile inland hex map.
  • Four local hotseat players.
  • Capital placement phase.
  • Colony placement phase.
  • Gameplay phase with turn advancement.
  • Season advancement after Player 4.
  • Basic income collection.
  • Resources: wood, stone, gold, food, influence, unrest.
  • Settlement pops: citizens, freemen, slaves.
  • Gameplay colony founding.
  • Colony-to-city upgrades with city adjacency validation.
  • Population capacity and overcapacity unrest.
  • Shared colony tiles with split income.
  • Basic buildings with data-driven costs and effects.
  • Player holdings panel and action log modal.

Missing or Deferred

  • Online multiplayer rooms and server authority.
  • Saved games and reconnect flow.
  • Assembly voting and resolutions.
  • National ideas and modifiers.
  • Luxury goods, trade, and advanced economy.
  • Temporary scoring and end-of-prototype winner display.
  • Conquest and displacement edge cases beyond shared-colony removal.
  • Unrest event deck and food-loss consequences.
  • Final victory conditions and endgame scoring.
  • Full UI art direction and final visual system.

How to Run

  • Install packages with npm install.
  • Start the dev server with npm run dev.
  • Run type checks with npm run check.
  • Run a production build with npm run build.

Current UI Guidance

Raise UI grievances now when they affect comprehension, testing, turn flow, or the ability to inspect rules. Save broad aesthetic overhaul work for after more Phase 1 and Phase 2 mechanics settle, so the final layout is designed around the real game surface.

A later dedicated UI overhaul is reasonable. The current UI should stay functional, readable, and easy to modify while the rules are still changing.
